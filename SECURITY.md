# Security Policy

## Supported Versions

Security fixes are provided for the latest main branch.

## Reporting a Vulnerability

Please do not open public issues for suspected security vulnerabilities.

Preferred process:

1. Open a private vulnerability report in the repository Security tab.
2. Include reproduction steps, impact, and any logs/screenshots that help triage.
3. Allow maintainers time to investigate before public disclosure.

If private reporting is not available, contact the repository owner directly and include "security" in the subject.

## Response Targets

- Initial triage acknowledgement: within 3 business days
- Follow-up status update: within 7 business days
- Fix timeline: depends on severity and complexity
